The missing letters spell out the clue phrase HASH OF FIRST COMMIT OF JSON FILES. Interpreting this correctly requires some technical knowledge. Each module has a JSON file containing information shown by the KTANE repository website (such as author, publication date, etc.). These JSON files are held in a git repository that is also hosted on GitHub called KtaneContent. Git keeps a record of all changes to the files, including when they were first added, and these changes have a 40-digit hexadecimal “hash” that identifies them. The clue phrase is telling us to look through the history of git changes in KtaneContent and find the hashes of those changes where the JSON file for each of these modules was added.

One way of obtaining these hashes is to navigate to the relevant JSON file on GitHub and clicking “history” in the top-right. For the first one, Regular Hexpressions, this takes us to the commit history for Regular Hexpressions.json. At the bottom of the list of changes we find the change where the file was added, and we can use the Copy button to obtain the full 40-digit hash.

Another way of obtaining the same hash is on the Linux command-line. Assuming a checkout of KtaneContent is available locally, you can navigate to the JSON folder and issue the following command-line:

git log --pretty=format:%H "Regular Hexpressions.json" | tail -n 1

Since we want to obtain this for all of the modules in this puzzle, the following more elaborate bash command can automate this:

for i in 'Regular Hexpressions' 'Beans' 'Faulty Sink' [etc.]; do echo $i `git log --pretty=format:%H "$i.json"|tail -n 1`; done

Once obtained, we can use the indexes provided in the puzzle to extract two-digit numbers, which in turn can be converted to letters using A1Z26.
