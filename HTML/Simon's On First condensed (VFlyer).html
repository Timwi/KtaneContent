On the Oversimplification of Simon’s On First

I know it’s supposed to flash that colour! I’m trying to describe this to you and I don’t understand if you are picking it up or not!

  • Refer to the original manual for the instructions.
  • This manual focuses on a major flaw with the tables that are used to calculate the referred colour alongside adding in colour references in a case where the user needs to cross reference the module and the manual.
  • On the module, each colour will have a number associated with it. These will change with each strike or stage progression.
  • Pressing a colour associated with an incorrect number will reset all inputs for the current stage and cause a strike. The stage progress will not reset.
  • A short delay where the module cannot be interacted with will follow each strike or stage completion on this module.

Condensed Portion

Because the original set of tables has a major flaw with how the tables are constructed, this procedure will always work.

Split into groups of 3 flashes and work out each group individually.

Take the numerical value of each flash in that group based on the colour it flashed.

Colour FlashedRedBlueLight GreenDark GreenOrangePinkPurpleYellow
Numerical Value01234567

Take the sum of the values in that group and modulo the sum by 8. The target colour is the colour whose numerical value is the result after modulo 8.

On the module find the colour that is directly opposite to the target colour to obtain the code.

Colour Opposite to Target ColourYellowDark GreenPinkRedLight GreenOrangeBluePurple
Code127284356418531675762843

Repeat the entire procedure for the other groups. This gives you the complete code for this stage. Press these numbers in order while respecting the labels shown on the module.
