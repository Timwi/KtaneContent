モジュール詳細：偽ボタン

ボタンを押すのはとても簡単だと思うかもしれない。そういう考え方こそ人々を爆発させるのだ。

モジュールには大きなボタンがあり、左側にインジケーターがある。
下のチャートを使用し、ボタンの処理方法を決定する。

色覚異常サポートを適用していると、ボタンの色を示すパターンが表示されている。

ボタンを長押しする必要がある場合、「偽離すタイミング」を参照する。

ボタンを連打する必要がある場合、「偽連打回数」を参照する。

モバイルデバイスの場合は、右にスクロールすることで全体を見ることができる。

偽離すタイミング

偽連打回数

ボタンを連打し始めると、インジケーターに押した総数が表示される。ボタンの色に基づいて計算する必要のある式を決定する。結果が１０～９９の範囲外の場合は範囲内になるまで繰り返し７を加算・減算する。
必要な回数ボタンを押したら、回答が送信されるまで３秒間待機する。

ボタンの色式
赤 (a + 2b) - d
オレンジ (2b + 1) - g
黄 (2a + d) - c
緑 d + (2f - b)
シアン (e + f + g) - b
青 2c + (d - 1)
紫 2(f - a) + d
ピンク 3g - (a + 3)
白 (f + ac) * (e + d)
黒 (ab + cd) - g(e - f)

変数一覧：

  • a = バッテリーの個数
  • b = ポートの種類の数
  • c = このモジュールを含む爆弾上の解除可能なモジュール数
  • d = インジケーターの数
  • e = シリアルナンバーの最後の数字*
  • f = シリアルナンバーの２番目の英字のアルファベット上の位置(A=1,B=2...)*
  • g = ボタンの単語の文字数

*シリアルナンバーに数字が含まれていないか、英字が２つ未満の場合、関連する値に０を使用する。
