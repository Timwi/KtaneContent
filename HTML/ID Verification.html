On the Subject of ID Verification

Hold on, didn’t we see this before? Must be a Wednesday thing.

Inspect the card on the module. If everything is correct and the card is valid, press the green button. Otherwise, press the red button.

There may be 2-4 cards. A new card will be shown upon a correct button press.

Pressing an incorrect button will register a strike and entire reset All cards will be removed and a new one will be shown.

There are 3 main things you must inspect:

  1. Card Formatting
  2. Employee information
  3. Card ID Code

Card Formatting

Information shown in order from top to bottom:

  • Employee name
  • Department
  • A 7-digit Employee ID
  • The employee’s email address
  • Date of Birth
  • Date of issue
  • Expiration Date
  • The 20-digit Card ID code, split into groups of 5 digits

  • Unusual misspellings
  • All dates should be written in DD/MM/YYYY format.
  • The Date of issue and the Expiration Date must be between 256 and 986 days apart.
  • The logo in the top right corner.

Card ID Code

The Card ID Code is implemented to help check the authenticity of the card. Make sure the Card ID Code satisfies all criteria listed below.

Checksums:

  • The last digit of the first three blocks of 5 digits must equal the least significant digit of the sum of the first four digits of that block.
  • The sixteenth digit of the code must equal the least significant digit of the sum of all the digits in the Employee ID.
  • The last digit of the code must equal the least significant digit of the sum of the first 19 digits.
