モジュール詳細：色コード

ロックにはコードが使用されている！何故色にしない？

コードは４桁と３色で構成されている。マニュアルの各セクションには、コードに含まれる数字または色が含まれている。

数字１

上から見て該当する最初の条件に従う。

  • 3：バッテリーが１個以下
  • 6：点灯したFRKインジケーターがある
  • 7：ポートの数がバッテリーの数より多い
  • 9：点灯したインジケーターの数が解除したモジュールの数より多い
  • 2：インジケーターとポートの総数がモジュール「色コード(Colour Code)」の数より少ない
  • 5：バッテリーの数がモジュール「Planets」の数より少ない
  • 8：４０を超えるモジュールがまだ解除されていない
  • 1：ちょうど２つの単３バッテリー及びちょうど２つの単１ばってりーがある
  • 4：解除されたモジュールが半分以上
  • 0：上記のいずれにも当てはまらない

数字２

背景の色に一致するルールを見て当てはまる条件に従う。

背景色 条件
赤 パラレルポートがある場合、数字は5である。
それ以外の場合、数字は3である。
オレンジ バッテリーが《インジケーター－ポート》よりも多い場合、数字は9である。
それ以外の場合、数字は4である。
緑 点灯したインジケーターの数がモジュール「Planets」の数よりも多い場合、数字は8である。
それ以外の場合、数字は1である。
黄 未解除のモジュール「色コード(Colour Code)」と「Planets」の総数が、解除済のモジュール「色コード(Colour Code)」と「Planets」の総数よりも多い場合、7.
それ以外の場合、数字は2である。
青 このモジュールが唯一の未解除モジュールの場合、数字は6である。
それ以外の場合、数字は0である。

数字３

  • バッテリーの数に２を加算し、解除済みのモジュールの数で乗算する。
  • 点灯したインジケーターが点灯していないインジケーターより多い場合、１５を減算する。
  • モジュールの背景が赤の場合、１５０を加算する。
  • 数が３の倍数の場合、３で除算する。
  • 数字１が０の場合、２倍にする。
  • 数字２が０の場合、４を乗算する。
  • 最下位桁を使用する。
  • 結果の絶対値が数字３の値である。
  • カウントダウンタイマーの最後の数字が数字３と等しいときに数字３を入力する。

数字４

  • 特定の数字を想定する。
  • 消灯インジケーターの数を減算する。
  • 未解除モジュール(特殊モジュールを除く)とモジュール「色コード(Colour Code)」の総数を加算する。
  • 数字１～３を合計して、それを加算する。
  • 結果が１００になる。

想定した数字の最下位桁が数字４である。

色１

背景色のルール内の該当する条件に従う。条件に合わない場合は紫を使用する。

背景色 条件
赤 ポート、インジケーター、解除済モジュールが全て無い場合、色は赤である。
それ以外の場合、色は紫である。
オレンジ シリアルナンバーの数字の合計の最下位桁がバッテリーの数と等しい場合、色はオレンジである。
それ以外の場合、色は紫である。
緑 ポートの数が点灯していないインジケーターの数よりも多い場合、色が緑である。
それ以外の場合、色は紫である。
黄 ちょうど１つの点灯していないインジケーターがあり、シリアルナンバーの最後の数字が奇数の場合、色は黄である。
それ以外の場合、色は紫である。
青 バッテリーの数が解除されたモジュールの数と等しい場合、色は青である。
それ以外の場合、色は紫である。

色２

色２ 条件
青 シリアルナンバーの最後の数字が偶数
緑 パラレルポートがある
オレンジ バッテリーの数にシリアルナンバーの数字の合計の最下位桁を加算したものが５以下である。
赤 単３バッテリーしかなく、少なくとも１つのバッテリーがある
黄 背景色が黄
紫 上記のいずれにも該当しない

色３

  • 未解除のモジュールの数に解除済モジュールの数を乗算する。
  • ３の倍数の場合、３で除算する。
  • 最下位桁を取る。
  • 色１が紫の場合、２で乗算する。
  • 色２が紫の場合、４で乗算する。
  • 最下位桁をとる。
  • バッテリーの数を乗算する。
  • ０～５の範囲になるまで繰り返し６を減算する。
  • 数字を色に変換する。
0 1 2 3 4 5
オレンジ 青 赤 紫 黄 緑

重要事項

コードにちょうど１つの０とちょうど１つの紫がある場合、カウントダウンタイマーの秒の２桁が０４または４０のときに送信を押す必要がある。

順序

上から順に見て該当する条件のタイプを順に使用し、次に下から上の順で該当しない条件のタイプを順に使用する。

タイプ 条件
数字 バッテリーの数が、現在の月の数字より多い
色 コード内の数字(０を除く)の積がモジュールの総数の最下位桁より多い。
数字 これが唯一の「色コード(Colour Code)」である。
色 現在時間が午前３時～午後４時の範囲内である。
数字 モジュールの総数が１０１または８１である。
色 「色コード(Colour Code)」モジュールの数が、モジュールの総数の半分の平方根を超える。
数字 シリアルナンバーにちょうど英字が３つある。
