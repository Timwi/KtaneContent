    begin with I = 30 // I = 20 // I = 0.
  2. Add 70 // 35 // 5 if button text CONTAINS same colour name // complementary colour name // any colour or "COLOUR".

  4. Add 15 for each display text that is the same colour as the button, and add 10 for each complementary colour.

Section 3: How to use the manual manual

A. Overview

This manual assumes that you have knowledge on terms used for this module. If one part is not understood, either refer back the original manual or several other alternative manuals.

Page 1 contains tables and their relevant uses, which are:

  • Glyphs table (C values calculation)
  • Button value calculation table (Final button values, X values)
  • Color table (Complementary colors)
  • Value X table (X values calculation)
  • Appended symbols table (appended symbols of displays 2,4,6,7,8 decryption, and button press timings)
  • Stage 4 symbol table (to determine which symbol for final stage)

Page 2 contains tables and their relevant uses, which are:

  • Alphabet position difference (AP) table (Text finding and B values calculation)
  • 5A column (A values calculation)
  • Display text color lookup table (R values calculation)
  • E trick (to Displays 1,3,5 text)

B. The E trick

The 3rd letter in displays 1,3,5 will be always be E and it is always located before a symbol. Based on whether the letters are identical (shown in red), the word can be easily determined, as well as their corresponding values.

For example, the second word E_XEEX is LETTER, E_XXXXXE is PHRASE, E_XXXXXX is COLOUR.

After obtaining the second word, the first word can also be determined in the same way using letters E,H,T and O.

C. The AP table

The original instruction can be found in Page 3 of this manual. This manual adds several extra optimization on top. Those includes marking AP between symbols grey, and unique and easily spottable text with blue.

D. A value calculations

Based on the AP determined and its difference, A value can be easily calculated by using the position of letter/symbol after encryption, minus its position before encryption, modulo the length of string.
It is reminded that the position of leftmost letter must be used to calculate the A value.

For example: display text is ECHO if unencrypted, and the AP is 2 and 26 (BZ) and the difference is 24, the position of the correct position is n, which is 1, from the AP table.
While the position of the symbol which has an AP of 2 is located in 4th (excluding appended symbols)(Z__B). Hence A=(4-1)%4 = 3. (the word has been shifted 3 times to the left during encryption)
