À propos du Binaire Cruel

Seulement 50% de chances d'échouer ! Oh attendez... En réalité, il y a 99.6% de chances.

Ce module contient cinq boutons libellés « 0 », « 1 », « R », « Send », « Read » et un écran. L'écran affiche le texte « 1 Message ». Appuyer le bouton « Read » affichera un mot sur l'écran. Puis, basé sur le mot affiché, calculer un code binaire à 8 chiffres et le valider. Le code binaire peut être calculé en utilisant les étapes ci-dessous. Le mot « Above » servira d'exemple.

Si les boutons ne sont pas rouges ou que l'écran n'affiche pas de mot, il s'agit d'un module différent.

  • Étape 1 : Convertir le mot obtenu en morse en utilisant l'Alphabet morse (Voir l'Annexe C01).

    (Exemple : Above -> .--...---...-.).

  • Étape 2 : Remplacer les tirets par des 1 et les points par des 0.

    (Exemple : .--...---...-. -> 01100011100010).

  • Étape 3 : Le nombre total de chiffres doit être divisible par 8. Si ce n'est pas le cas, ajouter le nombre de zéros nécessaires à la fin du code.

    (Exemple : 01100011100010. Il y a 14 chiffres, deux zéros doivent être ajoutés. -> 0110001110001000).

  • Étape 4 : Prendre les 8 premiers chiffres et les convertir en décimal. Prendre les 8 prochains chiffres et effectuer la même opération, etc.

    (Exemple : 0110001110001000. Les 8 premiers chiffres sont 01100011 ce qui donne 99 en décimal. Les 8 prochains chiffres sont 10001000 ce qui donne 136 en décimal).

  • Étape 5 : Ajouter les résultats obtenus, modulo 256, puis convertir le résultat en binaire.

    (Exemple : 136 + 99 = 235, ce qui donne 11101011 en binaire).

  • Étape 6 : Enfin, valider la réponse.

    (Exemple : Les boutons « 0 » et « 1 » servent à entrer « 11101011 » et le bouton « Send » sert à valider la réponse, tandis que le bouton « R » est utilisé pour réinitialiser la réponse si une erreur est commise).

À noter que la réponse doit être entrée dans un format complet ! Par exemple, si la réponse obtenue est 110, le code 00000110 doit être validé.
